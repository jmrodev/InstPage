Modelado de Datos y Procesos

Fundamentos del Modelado

El análisis de sistemas utiliza dos tipos principales de modelos:

  • Modelo Lógico: Muestra "QUÉ" debe hacer el sistema sin preocuparse por la implementación física.
  • Modelo Físico: Describe "CÓMO" se construirá el sistema en la práctica.

Muchos analistas siguen el enfoque de "cuatro-modelos":

  1. Modelo físico del sistema actual
  2. Modelo lógico del sistema actual
  3. Modelo lógico del nuevo sistema
  4. Modelo físico del nuevo sistema

Este enfoque permite entender claramente el sistema existente antes de mejorarlo.

Diagramas de Flujo de Datos (DFD)

Los DFD son herramientas gráficas que muestran cómo los datos se transforman a través del sistema.

Importante: Los DFD muestran el movimiento de datos, no la lógica de programación o los pasos del procesamiento.

Componentes Principales de los DFD

1. Procesos

  • Transforman datos de entrada en salida con diferente contenido o forma
  • Se nombran con verbo + sustantivo (ej: "CALCULAR COMISIÓN")
  • Funcionan como "cajas negras" que ocultan la lógica interna
  • Se pueden "ampliar" para mostrar su funcionamiento interno en DFDs de mayor detalle

2. Flujos de Datos

  • Son las rutas por donde viajan los datos entre puntos del sistema
  • Se nombran con sustantivo + adjetivo (ej: "FACTURA DE PAGO")
  • Pueden representar un solo dato o un conjunto de ellos

3. Almacenamientos de Datos

  • Guardan datos que serán usados posteriormente por los procesos
  • Se nombran con sustantivo + adjetivo (ej: "PAGOS DIARIOS")
  • Siempre tienen al menos un flujo de entrada y otro de salida

4. Entidades Externas

  • Representan fuentes o destinos de datos fuera del sistema
  • También llamadas "terminadores", "orígenes" o "sumideros"
  • Se nombran en singular (ej: "CLIENTE")

Reglas de Conexión en DFD

Conexiones permitidas:

  • Proceso → Proceso
  • Proceso → Entidad Externa
  • Proceso → Almacenamiento de Datos

Conexiones NO permitidas:

  • Entidad Externa → Entidad Externa
  • Entidad Externa → Almacenamiento de Datos
  • Almacenamiento de Datos → Almacenamiento de Datos

Niveles de los DFD

Los DFD se crean en diferentes niveles de detalle:

1. Diagrama de Contexto (nivel más alto)

  • Muestra todo el sistema como un único proceso (Proceso 0)
  • Presenta los límites del sistema y sus interacciones con el mundo exterior
  • Solo contiene un proceso central, entidades externas y flujos de datos
  • No muestra almacenamientos de datos

2. Diagrama 0 (segundo nivel)

  • Ampliación del proceso 0
  • Muestra los principales procesos, flujos y almacenamientos de datos
  • Mantiene las mismas entidades y conexiones externas del diagrama de contexto

3. DFDs de Bajo Nivel (niveles adicionales)

  • Desglosan procesos complejos en procesos más simples
  • Utilizan técnicas de nivelado y balanceo:
    • Nivelado: Aumentar progresivamente el detalle hasta llegar a primitivas funcionales
    • Balanceo: Mantener consistencia entre los flujos de entrada/salida en los diagramas padre e hijo

Diccionario de Datos

Complementa los DFD documentando todos los detalles de:

Componente Elementos Documentados
Elementos de datos Nombre, alias, tipo, longitud, valores aceptados, procedencia, usuarios responsables, etc.
Registros Nombre, descripción, nombres alternativos, atributos, etc.
Flujos de datos Nombre, descripción, origen, destino, volumen y frecuencia, etc.
Almacenamientos Nombre, descripción, nombres alternativos, atributos, volumen y frecuencia, etc.
Procesos Nombre, descripción, número del proceso, etc.
Entidades Nombre, descripción, nombres alternativos, flujos de entrada y salida, etc.

Herramientas de Descripción de Procesos

Para documentar las primitivas funcionales (procesos que no se desglosan más) se utilizan:

  • Lenguaje estructurado: Subconjunto del lenguaje formal que describe los procesos lógicos
  • Tablas de decisión: Estructuras lógicas que muestran todas las combinaciones de condiciones y resultados
  • Árboles de decisión: Representación gráfica de condiciones, acciones y reglas

Diseño Modular

El diseño modular se basa en tres estructuras lógicas fundamentales:

  1. Secuencia: Ejecución paso a paso
  2. Selección: Bifurcaciones basadas en condiciones
  3. Iteración: Repetición basada en condiciones

Cada estructura lógica debe tener un solo punto de entrada y salida.
