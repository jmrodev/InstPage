Usando PHP con Visual Studio Code

Inicio / Desarrollo Web / PHP / VS Code

Bienvenido a esta guía para configurar y utilizar PHP de manera efectiva dentro de Visual Studio Code (VS Code), uno de los editores de código más populares y potentes para el desarrollo web moderno.

Preparación del Entorno

Requisitos Previos

Antes de empezar a escribir código PHP en VS Code, necesitas asegurarte de tener tanto PHP como el propio editor instalados en tu sistema. VS Code por sí solo es un editor de texto; requiere el intérprete de PHP para ejecutar y depurar tu código.

Esta sección te guiará a través de los pasos esenciales para tener tu entorno listo.

Instalación de PHP
🐘

Necesitas tener PHP instalado en tu sistema operativo (Windows, macOS, Linux). Puedes descargarlo desde el sitio oficial o usar gestores de paquetes.

¿Por qué es necesario?

VS Code no incluye PHP. El intérprete de PHP es el programa que realmente lee y ejecuta tus scripts .php. Sin él, solo podrías escribir código, pero no probarlo ni ejecutarlo.

# Verifica si PHP está instalado y su versión $ php -v PHP 8.1.10 (cli) (built: Aug 30 2022 18:11:12) (NTS) Copyright (c) The PHP Group Zend Engine v4.1.10, Copyright (c) Zend Technologies # Si no aparece nada o da error, necesitas instalarlo.

Recursos de Instalación:

  • Documentación Oficial de Instalación PHP
  • Para Windows: Usa XAMPP, WAMP, Laragon o descarga desde windows.php.net.
  • Para macOS: Usa Homebrew (brew install php) o MAMP.
  • Para Linux (Debian/Ubuntu): sudo apt update && sudo apt install php
Tarea:
  1. Verifica si tienes PHP instalado usando php -v en tu terminal.
  2. Si no lo tienes, instálalo siguiendo la guía para tu sistema operativo.
  3. Asegúrate de que el comando php funcione globalmente en tu terminal.
Instalación de Visual Studio Code
💻

VS Code es un editor de código fuente gratuito, ligero y extensible desarrollado por Microsoft. Es ideal para el desarrollo PHP gracias a su amplio soporte y ecosistema de extensiones.

¿Por qué usar VS Code?

Ofrece resaltado de sintaxis, autocompletado inteligente (IntelliSense), depuración integrada, control Git, terminal incorporada y miles de extensiones para personalizar tu flujo de trabajo. Es el estándar de facto para muchos desarrolladores web.

# Verifica si tienes VS Code instalado (comando 'code') $ code --version 1.78.2 b3e4e68a0bc097f0ae7907b217c1119af9e03435 x64 # Si no funciona, necesitas instalarlo o añadirlo al PATH.

Pasos:

  • Ve a la página oficial de VS Code.
  • Descarga el instalador adecuado para tu sistema operativo (Windows, macOS, Linux .deb/.rpm).
  • Ejecuta el instalador y sigue las instrucciones.
  • (Opcional pero recomendado) Asegúrate de marcar la opción "Agregar a PATH" durante la instalación en Windows para poder lanzar VS Code desde la terminal.
Tarea:
  1. Descarga e instala VS Code si aún no lo tienes.
  2. Abre VS Code y familiarízate con la interfaz básica.
  3. Prueba a abrir tu terminal y ejecutar code . dentro de una carpeta para abrirla en VS Code.

Configuración de VS Code para PHP

Extensiones y Configuraciones Clave

La verdadera potencia de VS Code para PHP proviene de sus extensiones. Estas añaden funcionalidades como mejor autocompletado, depuración avanzada, formato de código y detección de errores (linting).

Extensiones Esenciales para PHP
🧩

Instalar las extensiones adecuadas mejorará drásticamente tu productividad al desarrollar con PHP en VS Code.

¿Por qué instalar extensiones?

El soporte básico de PHP en VS Code es bueno, pero las extensiones lo llevan al siguiente nivel. Te ayudan a escribir código más rápido, con menos errores y facilitan la depuración y el mantenimiento.

Extensiones Recomendadas:

  • PHP Intelephense (o PHP IntelliSense): Proporciona autocompletado avanzado, navegación de código, formato y diagnóstico de errores. Es una de las más populares y completas.
    Búscala como: bmewburn.vscode-intelephense-client
  • PHP Debug: Permite integrar VS Code con Xdebug para depurar tu código PHP paso a paso, inspeccionar variables, etc. Fundamental para solucionar problemas.
    Búscala como: xdebug.php-debug
  • PHP DocBlocker: Facilita la creación de bloques de comentarios PHPDoc estándar.
    Búscala como: neilbrayfield.php-docblocker
  • Composer: (Si usas Composer) Proporciona comandos y ayuda para manejar dependencias PHP.
    Búscala como: ikappas.composer

Cómo Instalar Extensiones:

  1. Abre VS Code.
  2. Ve a la vista de Extensiones (icono de bloques en la barra lateral o `Ctrl+Shift+X`).
  3. Busca el nombre de la extensión.
  4. Haz clic en "Instalar".
  5. Puede que necesites reiniciar VS Code después de instalar algunas extensiones.
Tarea:
  1. Abre la pestaña de Extensiones en VS Code.
  2. Busca e instala "PHP Intelephense".
  3. Busca e instala "PHP Debug".
  4. Instala cualquier otra extensión que consideres útil (como PHP DocBlocker).
Configuración del Intérprete PHP
⚙️

Aunque a menudo VS Code detecta automáticamente tu instalación de PHP, a veces necesitas especificar manualmente la ruta al ejecutable de PHP, especialmente si tienes múltiples versiones instaladas.

¿Por qué configurar la ruta?

Las extensiones como Intelephense y PHP Debug necesitan saber dónde encontrar el intérprete de PHP para validar la sintaxis correctamente, ejecutar scripts y habilitar la depuración. Si VS Code no lo encuentra, muchas funcionalidades no estarán disponibles.

Cómo configurar la ruta (Settings JSON):

  1. Abre la Paleta de Comandos (`Ctrl+Shift+P` o `Cmd+Shift+P`).
  2. Escribe "Open User Settings (JSON)" y selecciónalo.
  3. Dentro del objeto JSON {}, añade o modifica las siguientes líneas:
{ // Ejemplo para Windows (ajusta la ruta si es diferente) "php.validate.executablePath": "C:/path/to/your/php/php.exe", "php.executablePath": "C:/path/to/your/php/php.exe", // Ejemplo para Linux/macOS (ajusta la ruta si es diferente) // A menudo '/usr/bin/php' o '/usr/local/bin/php' // "php.validate.executablePath": "/usr/local/bin/php", // "php.executablePath": "/usr/local/bin/php", // Configuración recomendada para Intelephense (opcional) "intelephense.environment.phpVersion": "8.1.0", // Pon tu versión // ... otras configuraciones ... }

Nota: php.validate.executablePath es para la validación integrada de VS Code, mientras que php.executablePath puede ser usado por otras extensiones. Es bueno configurar ambas.

Tarea:
  1. Encuentra la ruta exacta a tu ejecutable php (o php.exe en Windows). Puedes usar where php (Windows) o which php (Linux/macOS).
  2. Abre tus "User Settings (JSON)" en VS Code.
  3. Añade la configuración php.validate.executablePath apuntando a tu ejecutable PHP. Guarda el archivo.
  4. (Opcional) Configura intelephense.environment.phpVersion con tu versión de PHP.

El Flujo de Trabajo Básico

Ahora que el entorno está configurado, veamos cómo crear, escribir y ejecutar archivos PHP directamente desde VS Code.

Notarás que el código PHP (entre y ?>) se colorea de forma diferente al HTML. Si tienes Intelephense, empezarás a recibir sugerencias de autocompletado para funciones y variables PHP.

Ejecutando Scripts PHP
▶️

Hay varias formas de ejecutar tu código PHP: usando el servidor web incorporado de PHP para ver páginas en el navegador, o ejecutando scripts directamente desde la terminal de VS Code.

¿Cómo probar el código?

Necesitas una forma de ver el resultado de tu script. El servidor web es ideal para archivos que generan HTML (como el ejemplo anterior), mientras que la ejecución directa en terminal es útil para scripts de consola o tareas automatizadas.

Método 1: Servidor Web Incorporado de PHP

  1. Abre la terminal integrada de VS Code (`Terminal > Nuevo Terminal` o `Ctrl+`).
  2. Asegúrate de estar en la carpeta raíz de tu proyecto.
  3. Ejecuta el comando: php -S localhost:8000
  4. Abre tu navegador web y ve a http://localhost:8000 (o http://localhost:8000/index.php). Verás tu página renderizada.
  5. Para detener el servidor, vuelve a la terminal y presiona `Ctrl+C`.
# Iniciar el servidor web en el puerto 8000 $ php -S localhost:8000 PHP 8.1.10 Development Server (http://localhost:8000) started [Tue May 16 10:30:00 2023] PHP 8.1.10 Development Server (http://localhost:8000) started [Tue May 16 10:30:05 2023] [::1]:51234 Accepted [Tue May 16 10:30:05 2023] [::1]:51234 [200]: GET / [Tue May 16 10:30:05 2023] [::1]:51234 Closing

Método 2: Ejecución Directa en Terminal

  1. Abre la terminal integrada de VS Code.
  2. Ejecuta el comando: php nombre_del_archivo.php
  3. La salida del script (lo que imprima con echo, print, etc.) aparecerá directamente en la terminal.
# Ejecutar el script index.php directamente $ php index.php Página PHP de Prueba

Tarea:
  1. Usa la terminal integrada de VS Code para iniciar el servidor web de PHP con php -S localhost:8000.
  2. Abre http://localhost:8000 en tu navegador y verifica que la página se muestra correctamente.
  3. Detén el servidor (`Ctrl+C`).
  4. Ejecuta el script directamente usando php index.php y observa la salida HTML en la terminal.

Depuración (Debugging) con Xdebug

Encontrando y Corrigiendo Errores

La depuración es el proceso de encontrar y corregir errores (bugs) en tu código. Usar un depurador como Xdebug con VS Code te permite ejecutar tu código línea por línea, inspeccionar el valor de las variables en cualquier momento y entender exactamente qué está sucediendo.

Es mucho más eficiente que llenar tu código de echo o var_dump.

Instalación y Configuración de Xdebug
🐞

Xdebug es una extensión PHP que necesita ser instalada y configurada en tu archivo php.ini para que VS Code pueda conectarse a ella.

¿Por qué Xdebug?

Es el estándar de facto para la depuración en PHP. Proporciona la interfaz necesaria para que herramientas como VS Code controlen la ejecución del script y analicen su estado. Sin Xdebug (o una alternativa similar), la extensión "PHP Debug" de VS Code no funcionará.

Pasos Generales:

  1. Instalar Xdebug: La forma más fácil suele ser usar PECL (si está disponible): pecl install xdebug. Si no, descarga el .dll (Windows) o .so (Linux/macOS) precompilado desde el asistente de Xdebug o compílalo tú mismo.
  2. Localizar tu php.ini: Ejecuta php --ini en la terminal para ver qué archivo php.ini está usando tu PHP (puede haber uno para CLI y otro para el servidor web).
  3. Configurar php.ini: Abre el archivo php.ini y añade las siguientes líneas (asegúrate de que la ruta a xdebug.so o xdebug.dll sea correcta y ajusta las opciones según Xdebug v3+): 
; Para Xdebug 3+
[xdebug]
zend_extension=ruta/completa/a/xdebug.so  ; O xdebug.dll en Windows
xdebug.mode=debug                         ; Activa el modo depuración
xdebug.start_with_request=yes             ; Inicia la depuración en cada request (útil para empezar)
; Opcional: 'trigger' para iniciar solo con parámetro/cookie
; xdebug.start_with_request=trigger
xdebug.client_host=127.0.0.1              ; IP donde escucha VS Code
xdebug.client_port=9003                   ; Puerto estándar para Xdebug 3+ (antes era 9000)
xdebug.log=/tmp/xdebug.log                ; (Opcional) Para ver logs de Xdebug
  1. Reiniciar Servidor Web/PHP-FPM: Si estás depurando a través de un servidor web (Apache, Nginx), necesitas reiniciarlo para que cargue la nueva configuración de php.ini.
  2. Verificar Instalación: Ejecuta php -v. Deberías ver una línea que dice "with Xdebug v3.x.x". También puedes crear un archivo con y buscar la sección de Xdebug al verlo en el navegador.
Tarea:
  1. Usa el asistente de Xdebug copiando la salida de phpinfo() para obtener instrucciones personalizadas.
  2. Instala Xdebug siguiendo las instrucciones.
  3. Localiza y edita tu archivo php.ini principal para añadir la configuración de Xdebug 3.
  4. Verifica la instalación con php -v o phpinfo().
Usando el Depurador de VS Code
🔬

Una vez Xdebug está configurado y la extensión "PHP Debug" instalada, puedes iniciar sesiones de depuración desde VS Code.

¿Cómo funciona?

VS Code escuchará conexiones entrantes de Xdebug. Al ejecutar tu script (ya sea desde el navegador o la terminal, dependiendo de la configuración), Xdebug se conectará a VS Code, permitiéndote controlar la ejecución.

Pasos para Depurar:

  1. Crear Configuración de Lanzamiento (`launch.json`):
    • Ve a la vista "Ejecutar y Depurar" (icono de play con un bug en la barra lateral o `Ctrl+Shift+D`).
    • Haz clic en "create a launch.json file".
    • Selecciona "PHP" en el menú desplegable.
    • Esto creará un archivo .vscode/launch.json con configuraciones predeterminadas. La configuración "Listen for Xdebug" es la más común para depuración web.
  2. Establecer Puntos de Interrupción (Breakpoints):
    • Abre el archivo PHP que quieres depurar.
    • Haz clic en el margen izquierdo, junto al número de línea donde quieres que la ejecución se pause. Aparecerá un punto rojo.
  3. Iniciar la Escucha:
    • En la vista "Ejecutar y Depurar", asegúrate de que "Listen for Xdebug" esté seleccionado en el menú desplegable superior.
    • Haz clic en el botón verde de "Play" (Iniciar depuración) o presiona `F5`. VS Code empezará a escuchar en el puerto configurado (por defecto 9003).
  4. Ejecutar el Script:
    • Abre tu script PHP en el navegador (si estás depurando una aplicación web y usas `xdebug.start_with_request=yes` o `trigger`).
    • O ejecuta el script desde la terminal (`php tu_script.php`), si Xdebug está configurado para CLI.
  5. Depurar:
    • La ejecución se detendrá en el primer punto de interrupción que encuentre.
    • Usa los controles de depuración en VS Code (Continuar F5, Saltar F10, Entrar F11, Salir Shift+F11).
    • Inspecciona las variables locales, globales y superglobales en el panel izquierdo.
    • Evalúa expresiones en la "Consola de Depuración".
  6. Detener la Depuración:
    • Haz clic en el botón rojo de "Stop" en la barra de herramientas de depuración.

Ejemplo .vscode/launch.json:

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Listen for Xdebug",
            "type": "php",
            "request": "launch",
            "port": 9003 // Asegúrate que coincida con php.ini
        },
        {
            "name": "Launch currently open script",
            "type": "php",
            "request": "launch",
            "program": "${file}",
            "cwd": "${fileDirname}",
            "port": 9003,
            "runtimeExecutable": "/ruta/a/tu/php" // Opcional si está en el PATH
        }
    ]
}
Tarea:
  1. Abre tu proyecto PHP en VS Code.
  2. Ve a la vista "Ejecutar y Depurar" y crea un archivo launch.json para PHP si no existe.
  3. Abre tu archivo index.php y pon un punto de interrupción en la línea $fechaActual = date('Y-m-d H:i:s');.
  4. Inicia la escucha de Xdebug (F5).
  5. Recarga la página http://localhost:8000 en tu navegador (asumiendo que el servidor PHP está corriendo y Xdebug configurado para iniciar con la petición).
  6. La ejecución debería pausarse en VS Code. Explora los paneles de Variables y la barra de herramientas de depuración (F10 para pasar a la siguiente línea, F5 para continuar hasta el final).

Características Útiles Adicionales

Optimizando tu Flujo de Trabajo

VS Code ofrece muchas otras características que, combinadas con las extensiones adecuadas, pueden hacer tu desarrollo PHP aún más eficiente.

IntelliSense y Autocompletado
💡

Gracias a extensiones como PHP Intelephense, obtienes sugerencias inteligentes mientras escribes, incluyendo funciones PHP, métodos de clase, variables, constantes y palabras clave.

¿Beneficios?

Reduce errores tipográficos, acelera la escritura de código y te ayuda a descubrir funciones o métodos sin tener que consultar constantemente la documentación.

  • Autocompletado de funciones nativas y definidas por el usuario.
  • Sugerencias de parámetros de funciones/métodos.
  • Navegación rápida a la definición de funciones/clases (`F12` o `Ctrl+Click`).
  • Detección de errores de sintaxis y tipos básicos en tiempo real.
Formateo y Linting
🧹

Mantén tu código limpio y consistente. VS Code puede formatear automáticamente tu código PHP según estándares definidos (como PSR-12) y detectar posibles problemas (linting).

¿Por qué es importante?

Un código bien formateado es más fácil de leer y mantener. El linting ayuda a identificar errores potenciales o malas prácticas antes de ejecutar el código.

  • Formateador: PHP Intelephense incluye un formateador. Puedes configurar VS Code para formatear al guardar (`editor.formatOnSave`). Presiona `Shift+Alt+F` para formatear manualmente.
  • Linter: VS Code y PHP Intelephense realizan linting básico. Para análisis más profundos, considera herramientas externas como PHPStan o Psalm e intégralas mediante extensiones.
Tarea:
  1. Asegúrate de tener `editor.formatOnSave` activado en tu configuración de VS Code (Settings JSON o UI).
  2. Desordena intencionalmente la indentación de tu archivo `index.php`.
  3. Guarda el archivo (`Ctrl+S`) y observa cómo se formatea automáticamente (si tienes Intelephense como formateador por defecto para PHP).
Integración con Git
🌿

VS Code tiene una excelente integración incorporada con el sistema de control de versiones Git.

¿Por qué usar Git?

Es esencial para el desarrollo moderno. Te permite rastrear cambios, colaborar con otros, revertir errores y gestionar diferentes versiones de tu proyecto PHP.

  • Inicializa repositorios (`git init`).
  • Visualiza cambios (añadidos, modificados, eliminados) en la vista de Control de Código Fuente (`Ctrl+Shift+G`).
  • Realiza `git add`, `git commit`, `git push`, `git pull` directamente desde la interfaz.
  • Maneja ramas (crear, cambiar, fusionar).
  • Visualiza diferencias (diffs) entre versiones.
Tarea (si usas Git):
  1. Asegúrate de tener Git instalado en tu sistema.
  2. Abre tu proyecto PHP en VS Code.
  3. Ve a la vista de Control de Código Fuente. Si no es un repositorio Git, haz clic en "Initialize Repository".
  4. Realiza algunos cambios en tu archivo `index.php`.
  5. Observa los cambios en la vista de Git. Haz "Stage Changes" (+) y escribe un mensaje de commit, luego confirma (✓).
